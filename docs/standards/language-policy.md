# Kebijakan Bahasa Dokumentasi

- Prose baru atau yang ditulis ulang menggunakan Bahasa Indonesia.
- Existing ADR dapat tetap menggunakan English untuk menjaga decision history.
- Code identifiers, filenames, paths, commands, routes, fields, roles, enums, environment variables, dan error codes harus ditulis persis seperti implementasi.
- Istilah teknis English dapat dipertahankan ketika terjemahan akan mengaburkan kontrak atau meaning.
- Satu section harus konsisten dan tidak mencampur bahasa tanpa kebutuhan teknis.
- Archive mempertahankan bahasa asli dan bukan acuan aktif.
