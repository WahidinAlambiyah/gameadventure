# Dokumentasi Proporsional

- Bug kecil memerlukan root cause, rencana perbaikan minimal, dan regression test; tidak otomatis memerlukan PRD.
- Feature kecil memerlukan scope dan acceptance criteria yang jelas.
- Feature besar memerlukan konteks produk, arsitektur/data/API yang relevan, serta test plan; buat ADR bila keputusan arsitektur berubah.
- Perbarui dokumentasi kanonis yang terdampak sebagai bagian Definition of Done.
- Jangan membuat dokumen baru bila dokumen kanonis yang ada dapat diperbarui.
