# Workflow Phase Closeout

1. Cocokkan seluruh acceptance criteria dengan implementation evidence.
2. Verifikasi focused tests, required CI, dan manual validation; catat gap secara jujur.
3. Pastikan PR telah direview dan merged tanpa scope drift atau protected-file accident.
4. Perbarui roadmap dan feature status kanonis dengan status implemented, partial, placeholder, atau planned.
5. Catat limitation, deferred work, security/data risk, dan dokumentasi yang belum lengkap.
6. Bersihkan branch hanya melalui tindakan git yang disetujui.
7. Identifikasi candidate phase berikutnya tanpa memasukkannya ke scope phase yang ditutup.
