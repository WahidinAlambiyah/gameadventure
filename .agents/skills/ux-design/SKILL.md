# UX Design

## Purpose

Menjaga pengalaman jelas, sesuai usia, accessible, dan konsisten dengan authority boundary.

## Use This Skill When

Mendesain child-facing UX, parent portal, loading/error/empty states, accessibility, atau interaction flow.

## Required Reading

Dokumen product/game-design yang relevan dan implementation UI saat ini.

## Responsibilities

Gunakan bahasa dan feedback sesuai usia, pisahkan konteks parent/child, sediakan accessible controls dan seluruh state penting.

## Mandatory Constraints

Jangan mendefinisikan ulang business rule atau memindahkan correctness, permission, completion, maupun unlock ke client.

## Expected Workflow

Identifikasi actor/task -> petakan states dan edge cases -> desain flow minimal -> cek accessibility dan authority.

## Validation

Review keyboard/focus/label/contrast serta manual responsive and interaction scenarios.

## Required Output

User flow, states, accessibility notes, authority boundary, dan validation plan/status.

## Out of Scope

Brand baru, design system generik, atau perubahan business policy.
